# 功能規格：圖片裁切與縮放編輯器

**功能分支**: `001-image-crop-resize`
**建立日期**: 2026-03-22
**狀態**: Draft
**輸入**: 建立一個類似 pintura 的前端圖片處理應用，包含 crop（含自動裁切）與 resize（含三種適配策略）功能

## 澄清記錄

### Session 2026-03-22

- Q: 裁切與縮放的操作是獨立還是可串聯？ → A: 可串聯，參考 pintura 作法：預設進入裁切模式，裁切完成後點擊工具列的 Resize icon button 進入縮放模式，兩者在同一工作階段內串聯執行
- Q: 輸出檔案格式為何？ → A: 提供格式下拉選單（PNG / JPEG / WebP），預設選項跟隨輸入圖片的格式；選擇 JPEG 時品質預設 92%
- Q: Contain 策略的背景填滿顏色為何？ → A: 提供顏色選擇器，預設為透明；輸出格式為 JPEG 時（不支援透明），自動改以白色填滿
- Q: 是否需要復原功能？ → A: 支援完整 Undo/Redo，工具列提供 Undo icon button 與 Redo icon button，同時支援 Ctrl+Z / Ctrl+Y 快捷鍵

## 用戶情境與測試（必填）

### 用戶故事 1 - 手動裁切圖片（優先級：P1）

使用者上傳一張圖片後，應用預設進入裁切模式。使用者透過互動式裁切工具拖動裁切框，選取想保留的區域。完成後可點擊工具列的 Resize icon button 切換至縮放模式，或直接下載裁切結果。

**為何此優先級**：裁切是最核心的操作，是整個應用的入口與基礎，其餘功能皆建立在此之上。

**獨立測試**：可透過上傳圖片 → 拖動裁切框 → 下載結果三個步驟完整驗證，不依賴 resize 或 auto-crop 功能。

**驗收情境**：

1. **Given** 使用者已上傳一張圖片，**When** 圖片載入完成，**Then** 應用應自動進入裁切模式，並在圖片上顯示初始裁切框（覆蓋全圖）
2. **Given** 使用者在裁切模式下，**When** 拖動裁切框的四個角落或四條邊的中間手把，**Then** 裁切框應即時更新，並顯示裁切後的預覽效果
3. **Given** 使用者已啟用長寬比鎖定（例如 16:9），**When** 拖動裁切框任意手把，**Then** 裁切框應維持該長寬比不變
4. **Given** 使用者已調整好裁切框，**When** 點擊下載按鈕，**Then** 系統應輸出裁切後的圖片並觸發下載
5. **Given** 使用者完成裁切，**When** 點擊工具列的 Resize icon button，**Then** 應用應切換至縮放模式，並以裁切後的結果作為縮放的輸入

---

### 用戶故事 2 - 自動裁切（優先級：P2）

使用者上傳一張有純色背景邊框的圖片（例如白邊截圖、掃描文件），點擊「自動裁切」按鈕後，系統自動偵測非背景內容的邊界，將裁切框設定至主體區域，使用者確認後可下載或繼續進行縮放。

**為何此優先級**：自動裁切建立在手動裁切的 UI 框架之上，需要 P1 先完成才能實作。

**獨立測試**：上傳一張四周有明顯純色邊框的圖片，點擊「自動裁切」，驗證裁切框是否自動收縮至非背景內容的邊界；再上傳無背景邊框的圖片，驗證邊界情況的提示。

**驗收情境**：

1. **Given** 使用者已上傳一張四周有明顯純色背景邊框的圖片，**When** 點擊「自動裁切」，**Then** 系統應將裁切框自動調整至非背景內容的最小邊界
2. **Given** 使用者觸發自動裁切，**When** 結果不理想，**Then** 系統應允許使用者手動微調自動產生的裁切框，並可重新確認
3. **Given** 使用者希望調整偵測靈敏度，**When** 調整容差滑桿，**Then** 裁切框應即時根據新容差值更新位置
4. **Given** 圖片整體像素顏色幾乎相同（無明顯背景與主體之分），**When** 點擊「自動裁切」，**Then** 系統應顯示提示訊息，告知無法偵測到有效主體範圍

---

### 用戶故事 3 - 調整圖片尺寸（優先級：P3）

使用者在裁切模式下完成裁切後，點擊工具列的 Resize icon button 切換至縮放模式。縮放的輸入來源為裁切後的結果（若未裁切則為原圖）。使用者指定目標寬高（像素），選擇一種適配策略（Contain / Cover / Fill），確認後取得符合目標尺寸的圖片。

**為何此優先級**：Resize 功能依賴裁切模式的 UI 框架，且為串聯流程的第二步；三種適配策略的視覺行為各異，需充分測試。

**獨立測試**：上傳圖片後直接點擊 Resize icon button（不裁切），輸入目標尺寸 800×600，分別選擇三種策略各別確認，驗證輸出圖片的尺寸與視覺結果符合各策略定義。

**驗收情境**：

1. **Given** 使用者點擊工具列的 Resize icon button，**When** 切換至縮放模式，**Then** 縮放的輸入應為當前裁切框內的圖片內容（若未設定裁切框則為原圖）
2. **Given** 使用者輸入目標尺寸並選擇「Contain」，**When** 確認縮放，**Then** 輸出圖片的長邊符合目標尺寸，短邊按等比例縮放，整張原圖完整可見，多餘空間以透明或指定背景色填滿
3. **Given** 使用者選擇「Cover」，**When** 確認縮放，**Then** 輸出圖片的短邊符合目標尺寸，長邊按等比例縮放，超出目標尺寸的部分被居中裁切，目標框完全被圖片填滿
4. **Given** 使用者選擇「Fill / Stretch」，**When** 確認縮放，**Then** 輸出圖片的寬高完全符合目標尺寸，允許長寬比改變
5. **Given** 使用者輸入的目標尺寸其中一個值為零或空白，**When** 嘗試確認縮放，**Then** 系統應顯示驗證錯誤，阻止操作進行

---

### 邊界情況

- 當自動裁切偵測到全圖皆為背景色時，裁切框保持原始大小並顯示說明提示
- 當裁切框被拖動至超出圖片邊界時，系統應限制裁切框不超出圖片範圍
- 當裁切框被縮至極小（寬或高接近零）時，系統應設定最小尺寸限制並阻止確認
- 當使用者未進行任何裁切直接切換至 Resize 模式時，縮放的輸入為原始圖片完整尺寸
- 當 Resize 目標尺寸與輸入圖片尺寸完全相同時，輸出應與輸入一致
- 上傳不支援格式的檔案時，系統應顯示錯誤提示並拒絕載入
- 當 Undo 堆疊為空時，按下 Ctrl+Z 或 Undo button 應無作用且不報錯
- 重新上傳圖片時，Undo/Redo 堆疊應完全清空

## 需求（必填）

### 功能需求

**通用**

- **FR-001**：系統 MUST 允許使用者透過本地檔案上傳圖片，支援 JPEG、PNG、WebP 格式
- **FR-002**：系統 MUST 採用 Pintura 風格的工具列模式切換：應用預設進入裁切模式；使用者點擊工具列的 Resize icon button 後切換至縮放模式；縮放模式的輸入來源為裁切後的結果（若未裁切則為原圖）
- **FR-003**：系統 MUST 提供下載按鈕與輸出格式下拉選單（PNG / JPEG / WebP），預設格式跟隨輸入圖片的格式；選擇 JPEG 時，需提供品質滑桿，預設品質為 92%；點擊下載後將最終結果以選定格式輸出至本地
- **FR-012**：系統 MUST 支援完整的 Undo/Redo 功能：工具列 MUST 顯示 Undo icon button 與 Redo icon button；同時支援 Ctrl+Z（Undo）與 Ctrl+Y（Redo）鍵盤快捷鍵；Undo/Redo 的操作範圍涵蓋裁切框調整、模式切換、縮放設定變更等所有使用者操作
- **FR-013**：當 Undo 堆疊為空時，Undo button MUST 顯示為停用狀態；當 Redo 堆疊為空時，Redo button MUST 顯示為停用狀態

**裁切（Crop）**

- **FR-004**：系統 MUST 在圖片預覽上顯示可互動的裁切框，框的四角與四邊中心各有拖動手把
- **FR-005**：系統 MUST 支援長寬比鎖定功能，可提供預設比例選項（1:1、4:3、16:9 等）及自由裁切模式
- **FR-006**：裁切框調整期間，系統 MUST 即時顯示裁切後的預覽效果
- **FR-007**：系統 MUST 提供自動裁切功能，算法行為如下：以圖片左上角像素的 RGB 值作為背景色基準，利用 RGB 三維空間的歐氏距離平方（避開開根號以提升效能）進行容差比對，從上、下、左、右四個方向逐行/逐列掃描，找出第一條包含非背景像素的邊界線，輸出裁切框的位置與尺寸
- **FR-008**：自動裁切功能 MUST 提供容差百分比調整控制項（範圍 0%–100%），調整時裁切框應即時更新

**縮放（Resize）**

- **FR-009**：系統 MUST 提供目標寬度與高度的數字輸入欄位（單位：像素），並驗證輸入值必須為正整數
- **FR-010**：系統 MUST 支援以下三種適配策略，行為與 CSS `object-fit` 屬性一致：
  - **Contain**：等比例縮放，圖片完整顯示於目標框內，空白區域以使用者選擇的背景色填滿（預設透明；輸出格式為 JPEG 時自動填白）；提供顏色選擇器供使用者自選背景色
  - **Cover**：等比例縮放，圖片完全覆蓋目標框，超出部分居中裁切
  - **Fill / Stretch**：強制填滿目標框，不保持長寬比，允許圖片變形
- **FR-011**：縮放 MUST 採用在處理速度與邊緣銳利度之間取得平衡的插值算法；預設使用 Bilinear（雙線性）插值，其品質優於 Nearest-Neighbor 且速度快於 Bicubic（雙三次）

### 關鍵實體

- **圖片（Image）**：使用者上傳的原始圖片，包含像素資料與原始寬高
- **裁切框（CropBox）**：定義裁切區域的矩形，屬性為 `x`、`y`、`width`、`height`（像素，相對於原始圖片）
- **縮放設定（ResizeConfig）**：目標寬度、目標高度、適配策略（contain / cover / fill）
- **自動裁切設定（AutoCropConfig）**：容差百分比（thresholdPercent，範圍 0–100）
- **編輯器狀態（EditorState）**：當前模式（crop / resize）、當前裁切框、當前縮放設定

## 成功標準（必填）

### 可量測成果

- **SC-001**：使用者從上傳圖片到下載最終結果，整體操作流程可在 60 秒內完成（不含使用者思考時間）
- **SC-002**：對四周有明顯純色背景的測試圖片集（10 張以上），自動裁切正確識別主體邊界的比例達 90% 以上
- **SC-003**：三種 Resize 適配策略的輸出尺寸與使用者指定目標尺寸完全一致（誤差為零像素）
- **SC-004**：三種適配策略的視覺結果可透過與 CSS `object-fit: contain/cover/fill` 行為的對比驗證，視覺上無可察覺差異
- **SC-005**：對 5MB 以內的圖片執行裁切或縮放操作，從點擊確認到結果可供下載，時間不超過 3 秒（在現代桌面瀏覽器中）
- **SC-006**：所有錯誤情境（不支援格式、無效尺寸輸入、無法偵測主體）均有明確的使用者提示，不出現靜默失敗
- **SC-007**：模式切換（裁切 → 縮放）操作回應時間不超過 300 毫秒

## 假設

- 本應用完全在瀏覽器前端執行，不需後端伺服器，圖片資料不會離開使用者裝置
- 自動裁切算法固定使用圖片左上角像素作為背景色基準（單點取樣），不支援多點平均取樣模式
- 插值算法預設使用 Bilinear（雙線性）插值；Bicubic 為可選升級項，不在 MVP 範圍內
- 應用介面語言為正體中文
- 使用者操作環境為現代桌面瀏覽器（Chrome 90+、Firefox 88+、Safari 14+）
- 不需支援多圖批次處理，每次操作僅處理單一圖片
- 使用者在同一工作階段中最多處理一張圖片（重新上傳即重置所有狀態）
